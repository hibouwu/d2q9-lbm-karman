## 1. 问题描述

`collision` 函数对网格内所有 `(width-2) × (height-2) = 800 × 158 = 126400` 个格点各自独立计算 BGK 碰撞算子，每个格点读写自己的 9 个分布函数，**无任何数据依赖**。

P2-1 修复循环顺序后，外层为 i（x 列），适合按列并行分配给不同线程：
- 每线程获得连续的列块，保证列内 j 循环的顺序访问不受干扰
- `schedule(static)` 静态分配，避免调度开销

### 1.1 并行安全性

- `mesh_out`：每格点独立写，无交叉写
- `mesh_in`：只读，无竞争
- `direction_matrix`、`equil_weight`：全局常量，只读
- 函数调用链（`compute_cell_collision` → `get_cell_density`、`get_cell_velocity`、`compute_equilibrium_profile`）：全局变量无修改，局部变量栈分配 → 完全线程安全

## 2. 修改内容

**文件：** `src/lbm/physics.cpp` → `collision`

```diff
+#pragma omp parallel for schedule(static)
 for (size_t i = 1; i < mesh_in->width - 1; i++) {
   for (size_t j = 1; j < mesh_in->height - 1; j++) {
     compute_cell_collision(...);
   }
 }
```
