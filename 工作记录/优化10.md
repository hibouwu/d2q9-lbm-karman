# 优化10 — `propagation` 从 push 模式改为 pull 模式（P2-2）

**日期：** 2026-04-23  
**分类：** P2 / 算法模式改写  
**状态：** 完成，正确性已验证

---

## 1. 问题描述

### 1.1 原始 push 模式

```cpp
for (size_t j = 0; j < mesh_out->height; j++) {
  for (size_t i = 0; i < mesh_out->width; i++) {
    for (size_t k = 0; k < DIRECTIONS; k++) {
      ssize_t ii = (i + direction_matrix[k][0]);  // 目标 x
      ssize_t jj = (j + direction_matrix[k][1]);  // 目标 y
      if (ii in bounds && jj in bounds) {
        mesh_out[ii][jj][k] = mesh_in[i][j][k];  // 写到目标
      }
    }
  }
}
```

push 模式的核心问题：
1. **写入随机目标**：`mesh_out[ii][jj]` 的位置由方向 k 决定，内层 k 循环对 mesh_out 的写入是非顺序的，引发 cache line 争用。
2. **无法并行化**：多个源格点可以写同一目标格点（例如同一列的两个相邻格点同时写目标的同一 ghost 列），存在写竞争，无法直接加 OpenMP。

### 1.2 push vs pull 等价性证明

push 写 `mesh_out[I][J][k] = mesh_in[I-dx][J-dy][k]` 的充要条件：`I-dx, J-dy ∈ [0,W)×[0,H)`。

pull 读：
```
mesh_out[I][J][k] = mesh_in[I-dx[k]][J-dy[k]][k]  当 I-dx,J-dy ∈ [0,W)×[0,H)
```

两者对每个 `(I,J,k)` 的判断条件完全相同，写入值也完全相同 → **数学等价**。

### 1.3 pull 模式优势

- **写入连续**：i-outer, j-inner，内层按 y 步进写 `mesh_out[i][j]`（连续内存），cache 友好。
- **每格点写一次**：输出格点 `(I,J)` 对 k=0..8 只被本格点自己的 pull 循环写，**无写竞争** → 安全并行化（P2-5 利用此特性）。

## 5. 结论

- push→pull 模式转换消除写竞争，改善写入的 cache 局部性
- np=1 提升 14%，np=2 提升 19%，np=4 提升 51%
- 为 P2-5 OpenMP 并行化奠定基础（pull 模式下每格点独立写入）
