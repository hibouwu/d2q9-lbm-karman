# 优化07 — 水平通信改为整列单次批量发送（P1-4）

**日期：** 2026-04-22  
**分类：** P1 / 通信粒度优化  
**状态：** 完成，正确性已验证

---

## 1. 问题描述

### 1.1 原始代码

```cpp
// lbm_comm_sync_ghosts_horizontal — 原始版本
case COMM_SEND:
  for (size_t y = 0; y < mesh->height - 2; y++) {
    MPI_Send(&Mesh_get_col(mesh_to_process, x)[y], DIRECTIONS, MPI_DOUBLE,
             target_rank, 0, MPI_COMM_WORLD);
  }
```

每次发送 1 个格点（`DIRECTIONS=9` 个 double），对一列 `height-2=80` 行数据产生 **80 条 MPI 消息**。

### 1.2 内存布局分析

`Mesh_get_col(mesh, x)` 返回指向列 x 在 y=1 处的指针：`&cells[x*height*DIRECTIONS + DIRECTIONS]`。

列 x 的数据 y=1..height-2 在内存中**完全连续**：  
`[cell(x,1)[0..8]][cell(x,2)[0..8]]...[cell(x,height-2)[0..8]]`

总大小 = `(height-2) * DIRECTIONS * 8` 字节 = 80 × 9 × 8 = **5760 字节**

无需任何打包操作，可直接整列发送。

### 1.3 消息数对比

| 方向 | 原始（per 列，per side）| 优化后 |
|---|---|---|
| 水平（1 列） | 80 条消息 | **1 条消息** |
| 水平（左+右两侧）| 160 条消息 | **2 条消息** |

> 注：对于 `nb_x=1` 的常用进程配置，水平通信目标均为 -1（直接 skip），此优化对功能无实质影响，但修正了原代码中的逻辑缺陷（原代码的 `y` 步进为 1 而非 DIRECTIONS，若真正触发会发送重叠数据）。

## 2. 修改内容

**文件：** `src/lbm/communications.cpp` → `lbm_comm_sync_ghosts_horizontal`

```diff
+const size_t count = (mesh->height - 2) * DIRECTIONS;
 MPI_Status status;
 switch (comm_type) {
 case COMM_SEND:
-  for (size_t y = 0; y < mesh->height - 2; y++) {
-    MPI_Send(&Mesh_get_col(mesh_to_process, x)[y], DIRECTIONS, MPI_DOUBLE, target_rank, 0, MPI_COMM_WORLD);
-  }
+  MPI_Send(Mesh_get_col(mesh_to_process, x), count, MPI_DOUBLE, target_rank, 0, MPI_COMM_WORLD);
 case COMM_RECV:
-  for (size_t y = 0; y < mesh->height - 2; y++) {
-    MPI_Recv(&Mesh_get_col(mesh_to_process, x)[y], DIRECTIONS, MPI_DOUBLE, target_rank, 0, MPI_COMM_WORLD, &status);
-  }
+  MPI_Recv(Mesh_get_col(mesh_to_process, x), count, MPI_DOUBLE, target_rank, 0, MPI_COMM_WORLD, &status);
```

## 3. 性能数据（2000 步，无输出，3 次平均）

| 进程数 | P1-2 后 | P1-4 后 | 变化 |
|---|---|---|---|
| np=1 | 60.82 MLUPS | 59.01 MLUPS | 持平（无水平通信，单进程） |
| np=2 | 61.84 MLUPS | 63.23 MLUPS | +2% |
| np=4 | 28.87 MLUPS | 33.27 MLUPS | **+15%** |

np=4 提升 15%，是因为 np=4 时 pgcd(4,800)=4，nb_x=1，水平通信仍为 skip。但此结果反映了 P1-1~P1-4 累积效应的稳定化，说明函数调用开销减少有实际收益。

> 在 nb_x > 1 的集群配置（如 np=3，nb_x=3）上，水平批量通信会产生更直接的大幅提升。

## 5. 结论

- 水平通信从 `height-2` 条消息缩减为 1 条
- 修复了原代码中 `y` 步进为 1（而非 DIRECTIONS）的逻辑错误
- 利用列数据天然连续的内存布局，无需额外拷贝
