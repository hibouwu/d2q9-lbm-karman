# 优化05 — 删除主循环 3 个 MPI_Barrier（P1-2）

**日期：** 2026-04-21  
**分类：** P1 / 同步开销消除  
**状态：** 完成，正确性已验证

---

## 1. 问题描述

`src/bin/main.cpp` 的时间步循环中有 3 个 `MPI_Barrier`：

```cpp
special_cells(...)
MPI_Barrier(MPI_COMM_WORLD);   // ← barrier 1

collision(...)
MPI_Barrier(MPI_COMM_WORLD);   // ← barrier 2

lbm_comm_halo_exchange(...)
propagation(...)
MPI_Barrier(MPI_COMM_WORLD);   // ← barrier 3
```

### 1.1 各 barrier 的冗余分析

| barrier | 位置 | 是否必要 | 理由 |
|---|---|---|---|
| barrier 1（special_cells 后）| `special_cells` 是纯本地计算，不涉及跨 rank 通信 | ❌ 多余 | 无跨 rank 依赖 |
| barrier 2（collision 后）| `collision` 同样是纯本地计算 | ❌ 多余 | 无跨 rank 依赖；`halo_exchange` 的阻塞通信已保证正确同步 |
| barrier 3（propagation 后）| `propagation` 是本地计算；`save_frame_all_domain` 内部用阻塞 `MPI_Recv/Send` 实现收集 | ❌ 多余 | `save_frame_all_domain` 自行同步 |

每次迭代多 3 次全局 barrier，20000 步 = **60000 次额外 MPI_Barrier**。

---

## 2. 修改内容

**文件：** `src/bin/main.cpp`

```diff
 special_cells(&mesh, &mesh_type, &mesh_comm);
-MPI_Barrier(MPI_COMM_WORLD);

 collision(&temp, &mesh);
-MPI_Barrier(MPI_COMM_WORLD);

 lbm_comm_halo_exchange(&mesh_comm, &temp);
 propagation(&mesh, &temp);
-MPI_Barrier(MPI_COMM_WORLD);
```

---

## 3. 性能数据（2000 步，无输出，3 次平均）

| 进程数 | P1-1 后基线 | P1-2 后 | 变化 |
|---|---|---|---|
| np=1 | 58.82 MLUPS | 60.82 MLUPS | +3% |
| np=2 | 62.21 MLUPS | 61.84 MLUPS | 持平 |
| np=4 | 25.30 MLUPS | 28.87 MLUPS | +14% |

np=4 提升 14%，说明主循环中 barrier 导致的跨 rank 等待在多进程时有显著开销。

---

## 4. 正确性验证

`python3.12 -m lbm_viz --check np1.raw np2.raw` → `ok`

---

## 5. 结论

- 每步减少 3 次全局同步
- 多进程场景收益更明显（np=4 提升 14%）
- 本地计算阶段之间无需全局屏障
